# 内容编写规范

## 基本原则

1. 学术中立：呈现主要解释路径与争议，不把单一立场写成定论。
2. 出处意识：只使用可公开验证的经典文本、通行教材与研究共识；不编造页码、引文或史料。
3. 术语统一：术语首次出现时给出简明定义；中英术语对照统一维护在 `docs/appendix/术语词典.md`。
4. 批判关系同步：正文出现明确批判关系时，同步更新 `data/critiques.json`。
5. 结构化数据同步：正文新增人物、流派、关键作品、核心概念时，同步更新 `data/philosophers.json` 或 `data/schools.json`。

## 标准章节模板

每个流派或思想运动文件使用以下二级标题：

```markdown
## 历史背景与产生原因
## 核心思想主张
## 主要代表人物
### 人物名
## 内部分歧与演化
## 对前人/其他流派的批判
## 后续流派对该学派的批判与继承
## 现代意义与延伸阅读
```

## 显式引用标记

为便于自动校验，可在正文中使用：

- `[[philosopher:康德]]`
- `[[school:德国唯心主义]]`
- `[[work:纯粹理性批判]]`
- `[[concept:先验综合判断]]`

脚本会检查人物与流派引用是否存在于 `data/*.json`。作品与概念目前只做格式提示，后续可扩展为强校验。

## 批判关系数据格式

`data/critiques.json` 中每条关系使用：

```json
{
  "source": "康德",
  "target": "休谟经验论",
  "critique_type": "认识论",
  "summary": "说明批判焦点与论证结构。",
  "key_text": "《纯粹理性批判》"
}
```

`critique_type` 限定为：`本体论`、`认识论`、`伦理学`、`政治哲学`、`美学`、`方法论`、`宗教哲学`、`社会哲学`、`语言哲学`、`科学哲学`、`综合`。

## 质量检查流程

每完成一个 docs 子目录：

```bash
python3 scripts/validate.py
python3 scripts/word_count.py
python3 scripts/build_index.py
```

通过后再提交 Git commit。commit message 使用中文，例如：

```text
新增启蒙运动章节初稿与批判关系数据
```
